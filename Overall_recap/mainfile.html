1. Variables

What is a Variable?

A variable is a container used to store data so that it can be reused and changed later.

Think of it like a box with a label. You can put things in (assignment) and take them out (usage).

let name = "Rubini";
const age = 25;

2. Data Types

JavaScript variables can hold different types of data. There are two main categories:

P Primitive Data Types

What are Primitive Types?

  • Immutable: Cannot be changed.
  • Stored by Value: Directly stores the data.
  • Not Objects: No methods (mostly).

NP Non-Primitive Data Types

What are Non-Primitive Types?

  • Reference Types: Variables store a reference (memory address), not the actual value.
  • Mutable: They can be changed after creation.
  • Complex: Can store multiple values or behaviors (Collections).
  • Heap Memory: Stored in heap memory, unlike stack for primitives.

üîç Stored by Reference (Important)

let arr1 = [1, 2];
let arr2 = arr1; // Reference copy

arr2.push(3);

console.log(arr1); // [1, 2, 3]

Because arr2 points to the same memory address as arr1, modifying one affects the other. This is different from primitive types!

What is a Conditional Statement?

A conditional statement is used to make decisions in a program.

  • ‚úî If true ‚Üí Code runs
  • ‚ùå If false ‚Üí Code is skipped or else block runs

Execution Flow

1Ô∏è‚É£ if Statement

Executes code only when the condition is true.

let age = 20;

if (age >= 18) {
  console.log("Eligible");
}
Flow: Check age >= 18 (True) ‚Üí Print Message

2Ô∏è‚É£ if...else Statement

Executes one block if true, another if false.

let marks = 40;

if (marks >= 50) {
  console.log("Pass");
} else {
  console.log("Fail");
}
Flow: Check marks >= 50 (False) ‚Üí Skip if ‚Üí Run else

3Ô∏è‚É£ for Loop

Repeats a block of code multiple times. Used when the number of repetitions is known.

for (let i = 1; i <= 5; i++) {
  console.log(i);
}

Execution Flow

  1. Initialize i = 1 (Runs once)
  2. Check i <= 5
  3. If true ‚Üí Run code
  4. Increment i++
  5. Repeat steps 2-4 until false
